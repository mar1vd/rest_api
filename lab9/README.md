# Lab 9: Load Testing with Locust

Це випробування API бібліотеки за допомогою Locust. Тестуємо один ендпоінт: `GET /books`.

## Як запустити

1. Встановіть Docker.
2. Перейдіть у папку `lab9`.
3. Запустіть:

```bash
docker compose up
```

4. Відкрийте в браузері:

```
http://localhost:8089
```

## Як використовувати

- Введіть кількість користувачів, наприклад `10`
- Введіть швидкість створення користувачів, наприклад `2`
- Натисніть `Start swarming`

## Що тестуємо

Locust виконує наступні дії:

1. Запитує токен через `POST /auth/token` з логіном `admin` і паролем `secret`
2. Виконує `GET /books` з заголовком `Authorization: Bearer <token>`

## Файли

- `locustfile.py` — сценарій Locust
- `docker-compose.yml` — піднімає Prism mock API та Locust UI
- `api-spec.yaml` — OpenAPI специфікація Prism mock сервісу

## Локальний тест

Після запуску сервісів у браузері можна побачити панель Locust та запустити навантаження. Якщо потрібно автоматично перевірити результат, використайте UI або команду Locust у headless-режимі.
